TVA NERC CIP-010 Configuration Change Management Summary

Standard: NERC CIP-010 - Configuration Change Management and Vulnerability Management
Applicability: High and Medium Impact BES Cyber Systems

1. CONFIGURATION CHANGE MANAGEMENT
TVA shall develop a baseline configuration for each BES Cyber System including:
- Operating system and version
- Any commercially available or open-source application software
- Custom software installed on the system
- Logical network accessible ports
- Security patches applied

Changes to baseline configurations must be:
- Authorized prior to implementation
- Documented within 30 days of completion
- Tested in a test environment when feasible before production deployment
- Reviewed for security impact before implementation

2. CONFIGURATION MONITORING
TVA shall monitor at least once every 35 calendar days for changes to the baseline configuration.
Unauthorized changes must be investigated and remediated within 35 days of detection.

3. VULNERABILITY MANAGEMENT
Vulnerability assessments required:
- High impact systems: at least once every 15 months
- Medium impact systems: at least once every 15 months
- Paper assessments permitted for systems that cannot be taken offline

Assessment results must be documented and remediation tracked to completion.

4. TRANSIENT CYBER ASSETS
Laptops and removable media used with BES Cyber Systems must:
- Have malicious code prevention software installed
- Have patches applied within 35 days
- Be managed per documented procedures before connection to BES Cyber Systems
